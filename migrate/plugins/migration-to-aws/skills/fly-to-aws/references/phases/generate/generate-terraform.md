# Generate Phase: Terraform Configuration Generation

> Loaded by `generate.md` when `aws-design.json` and `estimation-infra.json` exist AND compute routes include Terraform-targetable services.

**Execute ALL steps in order. Do not skip or optimize.**

## Overview

Transform `aws-design.json` into deployable Terraform HCL configurations for fly.io-to-AWS migrations. Produces a `terraform/` directory in `$MIGRATION_DIR/` containing valid, `terraform validate`-passing configurations for all designed AWS resources.

**NEVER create VPC/subnets/NAT/IGW resources** — use default-VPC fallback or VPC_ID/SUBNET_IDS variables (same guardrail as agent-advisor's poc-shapes).

## Prerequisites

Read the following artifacts from `$MIGRATION_DIR/`:

- `aws-design.json` (REQUIRED) — AWS architecture design from Phase 3
- `estimation-infra.json` (REQUIRED) — Cost estimates from Phase 4
- `preferences.json` (REQUIRED) — User migration preferences from Phase 2
- `fly-resource-inventory.json` (REQUIRED) — Discovered fly.io resources from Phase 1

If any required file is missing: **STOP**. Output: "Missing required artifact: [filename]. Complete the prior phase that produces it."

## Output Structure

Generate `$MIGRATION_DIR/terraform/` with the following file organization. Only emit domain files that have resources in `aws-design.json`:

| File           | Domain     | Contains                                              |
| -------------- | ---------- | ----------------------------------------------------- |
| `main.tf`      | core       | Provider config, backend, data sources                |
| `variables.tf` | core       | All input variables with types and defaults           |
| `outputs.tf`   | core       | Resource outputs and migration summary                |
| `network.tf`   | networking | VPC data sources (NO VPC creation), security groups   |
| `compute.tf`   | compute    | ECS cluster, Fargate task definitions, services, ALBs |
| `database.tf`  | database   | RDS/Aurora instances, parameter groups, RDS Proxy     |
| `cache.tf`     | cache      | ElastiCache replication groups, subnet groups         |
| `storage.tf`   | storage    | S3 buckets, EFS file systems (conditional)            |
| `lambda.tf`    | compute    | Lambda functions, function URLs, lifecycle hooks      |
| `security.tf`  | security   | IAM roles/policies                                    |

**File emission rules:**

- `main.tf`, `variables.tf`, `outputs.tf` — ALWAYS emitted
- `network.tf` — ALWAYS emitted (contains VPC data sources and security groups)
- `compute.tf` — Emitted when `compute.<group>.target` contains "fargate__" or "ecs__" entries
- `lambda.tf` — Emitted when `compute.<group>.target` contains "lambda" or "lambda_microvms" entries
- `database.tf` — Emitted when `databases[]` is non-empty
- `cache.tf` — Emitted when `cache[]` is non-empty
- `storage.tf` — Emitted when `storage[]` is non-empty
- `security.tf` — ALWAYS emitted (IAM roles required for all deployments)

**Service-to-file routing:**

| AWS Service in `aws-design.json`                | Target File   |
| ----------------------------------------------- | ------------- |
| Fargate, ECS, ALB                               | `compute.tf`  |
| Lambda, Lambda MicroVMs                         | `lambda.tf`   |
| RDS PostgreSQL, Aurora PostgreSQL, Aurora MySQL | `database.tf` |
| ElastiCache Redis, ElastiCache Serverless       | `cache.tf`    |
| S3, EFS                                         | `storage.tf`  |
| VPC data source, Security Groups, NACLs         | `network.tf`  |
| IAM Roles, IAM Policies                         | `security.tf` |

**Unmapped services:** If `aws-design.json` contains a compute target with no Terraform resource mapping in this file (e.g., `agentcore`, `bedrock_handoff`, unknown targets), **skip** that resource and log a warning to `generation-warnings.json`. Do NOT halt generation.

## Step 1: Generate `main.tf`

```hcl
# Fly.io-to-AWS Migration — Terraform Configuration
#
# Generated by the fly-to-aws migration skill.
# This configuration implements the architecture designed in aws-design.json.
#
# Apply sequence:
#   1. terraform init
#   2. terraform plan -out=tfplan
#   3. terraform apply tfplan

terraform {
  required_version = ">= 1.5.0"

  required_providers {
    aws = {
      source  = "hashicorp/aws"
      version = "~> 5.80"
    }
  }
}

provider "aws" {
  region = var.aws_region

  default_tags {
    tags = {
      Project     = var.project_name
      Environment = var.environment
      ManagedBy   = "terraform"
      MigrationId = var.migration_id
      Source      = "fly-to-aws"
    }
  }
}

data "aws_caller_identity" "current" {}
data "aws_region" "current" {}
data "aws_availability_zones" "available" {
  state = "available"
}
```

**Customization rules:**

- `region` value: Use `var.aws_region` (populated from `preferences.json.global.target_region`)
- `MigrationId` tag: Use the migration run ID from `.phase-status.json`

## Step 2: Generate `variables.tf`

**Always include these global variables:**

```hcl
variable "aws_region" {
  description = "AWS region for all resources"
  type        = string
  default     = "<preferences.json.global.target_region>"
}

variable "project_name" {
  description = "Project name used for resource naming"
  type        = string
  default     = "<fly_app_name or migration_id>"
}

variable "environment" {
  description = "Environment name (e.g., production, staging)"
  type        = string
  default     = "<preferences.json.global.environment_naming>"
}

variable "migration_id" {
  description = "Migration run identifier"
  type        = string
  default     = "<migration_id from .phase-status.json>"
}

# VPC configuration — NEVER create VPC/subnets/NAT/IGW
# Use default VPC or provide existing VPC ID/subnet IDs
variable "vpc_id" {
  description = "Existing VPC ID (or use default VPC)"
  type        = string
  default     = null  # null = use default VPC
}

variable "subnet_ids" {
  description = "Existing subnet IDs (or use default VPC subnets)"
  type        = list(string)
  default     = null  # null = use default VPC subnets
}
```

**Per-service variables** — Extract from `aws-design.json` for each designed service. Include:

- Compute: `container_image_*` (one per Fargate/ECS service), `desired_count_*`
- Lambda: `lambda_function_name_*`, `lambda_memory_*`, `lambda_timeout_*`
- Database: `db_instance_class`, `db_storage_gb`, `db_engine_version`, `db_multi_az`
- Cache: `cache_node_type`, `cache_engine_version`
- Storage: `s3_bucket_name_*`, `efs_storage_gb_*`

**Naming convention:** `<resource_type>_<fly_app>_<attribute>` (sanitize app names: replace `-` with `_`).

Use `aws_config` values from `aws-design.json` as defaults. Add fly.io source as comment:

```hcl
variable "fargate_cpu_my_web_app_web" {
  description = "Fargate CPU units for my-web-app web process"
  type        = number
  default     = 0.25
  # Fly source: shared-cpu-1x (0.25 vCPU)
}

variable "fargate_memory_my_web_app_web" {
  description = "Fargate memory GB for my-web-app web process"
  type        = number
  default     = 0.5
  # Fly source: shared-cpu-1x (512 MB)
}
```

## Step 4: Generate `network.tf`

**NEVER create VPC/subnets/NAT/IGW resources.** Use default VPC or data sources for existing VPC.

```hcl
# VPC — Use default VPC or existing VPC (NEVER create new VPC)
# Fly.io apps run in Fly's shared network. AWS migration uses default VPC or existing VPC.

data "aws_vpc" "selected" {
  id      = var.vpc_id != null ? var.vpc_id : data.aws_vpc.default[0].id
  default = var.vpc_id == null
}

data "aws_vpc" "default" {
  count   = var.vpc_id == null ? 1 : 0
  default = true
}

data "aws_subnets" "selected" {
  filter {
    name   = "vpc-id"
    values = [data.aws_vpc.selected.id]
  }
}

# Security Groups

# ALB Security Group (for web services)
resource "aws_security_group" "alb" {
  name_prefix = "${var.project_name}-${var.environment}-alb-"
  vpc_id      = data.aws_vpc.selected.id
  description = "Security group for Application Load Balancer"

  ingress {
    from_port   = 443
    to_port     = 443
    protocol    = "tcp"
    cidr_blocks = ["0.0.0.0/0"]
    description = "HTTPS from internet"
  }

  ingress {
    from_port   = 80
    to_port     = 80
    protocol    = "tcp"
    cidr_blocks = ["0.0.0.0/0"]
    description = "HTTP from internet (redirects to HTTPS)"
  }

  egress {
    from_port   = 0
    to_port     = 0
    protocol    = "-1"
    cidr_blocks = ["0.0.0.0/0"]
    description = "Allow all outbound"
  }

  tags = {
    Name = "${var.project_name}-${var.environment}-alb-sg"
  }

  lifecycle {
    create_before_destroy = true
  }
}

# Fargate Task Security Group
resource "aws_security_group" "fargate" {
  name_prefix = "${var.project_name}-${var.environment}-fargate-"
  vpc_id      = data.aws_vpc.selected.id
  description = "Security group for Fargate tasks"

  ingress {
    from_port       = 0
    to_port         = 65535
    protocol        = "tcp"
    security_groups = [aws_security_group.alb.id]
    description     = "Traffic from ALB"
  }

  egress {
    from_port   = 0
    to_port     = 0
    protocol    = "-1"
    cidr_blocks = ["0.0.0.0/0"]
    description = "Allow all outbound"
  }

  tags = {
    Name = "${var.project_name}-${var.environment}-fargate-sg"
  }

  lifecycle {
    create_before_destroy = true
  }
}

# Database Security Group (conditional — only if databases[] non-empty)
resource "aws_security_group" "database" {
  count       = <1 if databases[] non-empty, else 0>
  name_prefix = "${var.project_name}-${var.environment}-db-"
  vpc_id      = data.aws_vpc.selected.id

  ingress {
    from_port       = 5432  # PostgreSQL port (adjust per engine)
    to_port         = 5432
    protocol        = "tcp"
    security_groups = [aws_security_group.fargate.id]
    description     = "PostgreSQL from Fargate tasks"
  }

  egress {
    from_port   = 0
    to_port     = 0
    protocol    = "-1"
    cidr_blocks = ["0.0.0.0/0"]
    description = "Allow all outbound"
  }

  tags = {
    Name = "${var.project_name}-${var.environment}-db-sg"
  }

  lifecycle {
    create_before_destroy = true

}

# Cache Security Group (conditional — only if cache[] non-empty)
resource "aws_security_group" "cache" {
  count       = <1 if cache[] non-empty, else 0>
  name_prefix = "${var.project_name}-${var.environment}-cache-"
  vpc_id      = data.aws_vpc.selected.id
  description = "Security group for ElastiCache"

  ingress {
    from_port       = 6379  # Redis port
    to_port         = 6379
    protocol        = "tcp"
    security_groups = [aws_security_group.fargate.id]
    description     = "Redis from Fargate tasks"
  }

  egress {
    from_port   = 0
    to_port     = 0
    protocol    = "-1"
    cidr_blocks = ["0.0.0.0/0"]
    description = "Allow all outbound"

  tags = {
    Name = "${var.project_name}-${var.environment}-cache-sg"
  }

  lifecycle {
    create_before_destroy = true

```

**Security group rules:**

- Only emit security groups for services present in `aws-design.json`
- Fargate SG allows traffic from ALB SG only (not open CIDR)
- Database/Cache SGs allow traffic from Fargate SG only
- ALB SG allows 80 and 443 from 0.0.0.0/0
- All SGs allow all outbound (Fargate needs ECR, internet access)

```hcl
  tags = {
    Name = "${var.project_name}-${var.environment}-cluster"

```

### CloudWatch Log Groups (per Fargate service)

```hcl
resource "aws_cloudwatch_log_group" "<group_name_sanitized>" {
  name              = "/ecs/${var.project_name}-${var.environment}/<process_group>"

  }

```

### Fargate Task Definitions

Generate one task definition per process group:

```hcl
# Fargate Task Definition — <fly_app>:<process_group>
# Fly source: <vm.preset>, auto_stop=<auto_stop>, min_machines_running=<min>
resource "aws_ecs_task_definition" "<app_sanitized>_<process_group>" {
  family                   = "${var.project_name}-${var.environment}-<process_group>"
  network_mode             = "awsvpc"
  requires_compatibilities = ["FARGATE"]
  cpu                      = "<aws_config.cpu × 1024>"
  memory                   = "<aws_config.memory_gb × 1024>"
  execution_role_arn       = aws_iam_role.ecs_execution.arn
  task_role_arn            = aws_iam_role.ecs_task.arn

  container_definitions = jsonencode([{
    name  = "<process_group>"
    image = var.<container_image_variable>
    portMappings = <[{containerPort: internal_port, protocol: "tcp"}] for web services, [] for workers>
    stopTimeout = <kill_timeout from fly.toml or 30>
    logConfiguration = {
      logDriver = "awslogs"
      options = {
        "awslogs-group"         = aws_cloudwatch_log_group.<ref>.name
        "awslogs-region"        = var.aws_region
        "awslogs-stream-prefix" = "<process_group>"
      }
    }
    secrets = <[{name: "SECRET_NAME", valueFrom: "arn:aws:ssm:..."}] from secrets.store>
    essential = true
  }])

  tags = {
    Name         = "${var.project_name}-${var.environment}-<process_group>-task"
    FlyApp       = "<fly_app>"
    ProcessGroup = "<process_group>"
  }
}
```

**Task definition rules:**

- `cpu` and `memory` come from `compute.<group>.sizing` (from machine preset table)
- `stopTimeout`: Set from `kill_timeout` in fly.toml + SIGINT note if app traps SIGINT
- `portMappings` included only for web process types (from `services[].internal_port`)
- Workers, release, and custom process types: no `portMappings`
- Secrets via SSM `valueFrom` (from `secrets.store` in aws-design.json)
- Container image: use variable reference (placeholder image at generation time)
- **Bluegreen caveat**: If `flags.stateful_mounts[]` is non-empty, add comment: "Bluegreen deployment not supported on Fly when volumes attached — ECS rolling deployment used instead."

### Fargate Services

```hcl
# Fargate Service — <fly_app>:<process_group>
resource "aws_ecs_service" "<app_sanitized>_<process_group>" {
  name            = "${var.project_name}-${var.environment}-<process_group>"
  cluster         = aws_ecs_cluster.main.id
  task_definition = aws_ecs_task_definition.<app_sanitized>_<process_group>.arn
  desired_count   = <min_machines_running or 1>
  launch_type     = "FARGATE"

  network_configuration {
    subnets          = data.aws_subnets.selected.ids
    security_groups  = [aws_security_group.fargate.id]
    assign_public_ip = false
  }

  # Load balancer block included ONLY for web process types
  load_balancer {
    target_group_arn = aws_lb_target_group.<app_sanitized>_web.arn
    container_name   = "web"
    container_port   = <internal_port>
  }

  depends_on = [aws_lb_listener.https]

  tags = {
    Name         = "${var.project_name}-${var.environment}-<process_group>-svc"
    FlyApp       = "<fly_app>"
    ProcessGroup = "<process_group>"
  }
}
```

**Service rules:**

- `desired_count`: Use `min_machines_running` from inventory, or 1 if min=0 and layer_fired=4 (scale-to-zero → always-on)
- `load_balancer` block included ONLY when `services[]` is non-empty (web process types)
- Workers, release, and custom processes: omit `load_balancer` block and `depends_on`
- `assign_public_ip = false` — tasks run in private subnets (or default VPC private routing)

### Application Load Balancer (web process types only)

Generate ALB resources only when `services[]` contains HTTP handlers:

```hcl
# Application Load Balancer — <fly_app> web traffic
# Fly source: fly.io HTTP routing → ALB routing
resource "aws_lb" "<app_sanitized>_web" {
  name               = "${var.project_name}-${var.environment}-alb"
  internal           = false
  load_balancer_type = "application"
  security_groups    = [aws_security_group.alb.id]
  subnets            = data.aws_subnets.selected.ids

  tags = {
    Name   = "${var.project_name}-${var.environment}-alb"
    FlyApp = "<fly_app>"
  }
}

resource "aws_lb_target_group" "<app_sanitized>_web" {
  name        = "${var.project_name}-${var.environment}-tg"
  port        = <internal_port>
  protocol    = "HTTP"
  vpc_id      = data.aws_vpc.selected.id
  target_type = "ip"

  health_check {
    enabled             = true
    healthy_threshold   = 3
    unhealthy_threshold = 3
    timeout             = 5
    interval            = 30
    path                = "/"
    protocol            = "HTTP"
    matcher             = "200-399"
  }

  tags = {
    Name = "${var.project_name}-${var.environment}-tg"
  }
}

resource "aws_lb_listener" "https" {
  load_balancer_arn = aws_lb.<app_sanitized>_web.arn
  port              = 443
  protocol          = "HTTPS"
  ssl_policy        = "ELBSecurityPolicy-TLS13-1-2-2021-06"
  certificate_arn   = var.acm_certificate_arn

  default_action {
    type             = "forward"
    target_group_arn = aws_lb_target_group.<app_sanitized>_web.arn
  }
}

resource "aws_lb_listener" "http_redirect" {
  load_balancer_arn = aws_lb.<app_sanitized>_web.arn
  port              = 80
  protocol          = "HTTP"

  default_action {
    type = "redirect"
    redirect {
      port        = "443"
      protocol    = "HTTPS"
      status_code = "HTTP_301"
    }
  }
}

variable "acm_certificate_arn" {
  description = "ARN of the ACM certificate for HTTPS listener"
  type        = string
  # TODO: Provide your ACM certificate ARN
}
```

**ALB rules:**

- HTTP listener always redirects to HTTPS
- TLS 1.3 policy for new deployments
- Health check path defaults to `/` (user should customize)
- ACM certificate ARN as variable with TODO marker
- Shared ALB across ≤25 services (ECS Express Mode pattern); escape hatch: multiple ALBs for >25 services

### Lambda MicroVMs (SnapStart-like lifecycle)

For `lambda_microvms` routes, add lifecycle hook scripts to `$MIGRATION_DIR/scripts/`:

```bash
# scripts/microvm-lifecycle.sh
# /run /suspend /resume /terminate lifecycle-hook contract
# Suspend/resume parity via Lambda container image caching

# Lambda SnapStart is Java-only. For other languages, use container image caching:
# - Minimize cold start by optimizing image layers
# - Use Lambda provisioned concurrency for near-instant resume
# - Snapshot storage cost: $0.08/GB-mo (billed separately)

# Example: Python suspend/resume emulation
# 1. Package app state to /tmp/snapshot.pkl
# 2. Lambda freezes container after invocation
# 3. Resume: Lambda thaws container, reload from /tmp/snapshot.pkl
```

Add note to `MIGRATION_GUIDE.md` about Lambda MicroVMs parity and per-second billing + snapshot storage costs.

- Unknown compute targets not in the dispatch table
- Extensions not in fast-path table (from `specialist_gates[]`)

## Step 13: Validate Generated Configuration

After all files are written:

1. **Syntax check**: Verify all `.tf` files are syntactically valid HCL
2. **Reference integrity**: Ensure all `resource` references resolve to declared resources within the same configuration
3. **Variable completeness**: Every `var.*` reference has a corresponding `variable` block in `variables.tf`
4. **Output references**: Every `output` references a declared resource attribute
5. **Tag consistency**: Every resource has the default tags (applied via provider `default_tags`)

**Note:** Full `terraform validate` requires `terraform init` (provider download). The generated configuration SHOULD pass `terraform validate` when run with network access. If validation cannot run (no Terraform binary, no network), log a note but do NOT block generation.

## Completion Handoff Gate (Fail Closed)

Before returning control to `generate.md`, require:

1. `$MIGRATION_DIR/terraform/main.tf` exists
2. `$MIGRATION_DIR/terraform/variables.tf` exists
3. `$MIGRATION_DIR/terraform/outputs.tf` exists
4. At least one domain file (`compute.tf`, `lambda.tf`, `database.tf`, `cache.tf`, `storage.tf`, or `network.tf`) exists beyond core files
5. All resource cross-references resolve within the configuration
6. `generation-warnings.json` exists if any services were skipped (empty `warnings` array if all mapped successfully)

If this gate fails: STOP and output: "generate-terraform did not produce a valid terraform/ directory; do not continue Generate."
